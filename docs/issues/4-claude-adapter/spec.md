# Issue #4: 实现 ClaudeCodeAdapter (生成 CLAUDE.md)

## 1. 需求分析

### 1.1 背景

PGC 的核心价值链是：**PGCDocument (YAML) → Adapter → Runtime 配置文件**。ClaudeCodeAdapter 是第一个具体 Adapter 实现，将 PGC 治理契约翻译为 Claude Code 能原生理解并严格执行的 `CLAUDE.md` 规则文件。

### 1.2 目标

1. 继承 `BaseAdapter`，实现 `ClaudeCodeAdapter`
2. 将 `Persona` 映射为 `CLAUDE.md` 中的 `# Role & Responsibilities` 和 `# Absolute Boundaries`
3. 将 `GovernanceGate` + `GovernanceBinding` 映射为 `# Governance Rules`（明确说明违反规则时的预期行为，如 "STOP and ask for human review"）
4. (可选) 生成配套的 `mcp_tools_schema.json` 骨架，声明所需的 Capability

### 1.3 约束

- 输入必须是已通过验证的 `PGCDocument`
- 输出 `CLAUDE.md` 格式美观、指令清晰，能被 Claude Code 直接读取并作为系统约束
- 不做 IO 写入，只返回字符串映射
- 必须通过 `BaseAdapter` 的 `validate_compatibility()` 检查

## 2. 设计方案

### 2.1 映射规则

| PGC 实体 | CLAUDE.md 章节 | 映射逻辑 |
|----------|---------------|----------|
| `Persona` | `# Role & Responsibilities` | 每个 Persona 生成一个 `## {name}` 子章节，列出 responsibilities |
| `Persona.negative_boundaries` | `# Absolute Boundaries` | 每个 boundary 生成一条 `> NEVER: ...` 规则 |
| `GovernanceGate` + `GovernanceBinding` | `# Governance Rules` | 每个 binding 生成一条规则，包含 gate 描述、authority level 和 violation policy |
| `Persona.output_target` | `# Output Targets` | 每个 Persona 的交付物去向 |
| `Capability` | `# Capabilities` | 声明可用能力及其归属 |

### 2.2 CLAUDE.md 模板结构

```markdown
# CLAUDE.md

> Auto-generated by PGC (Persona-Governance-Capability) Adapter for Claude Code.
> DO NOT edit manually — regenerate from your .pgc.yaml source.

---

## Role & Responsibilities

### {persona.name} (`{persona.id}`)

- {responsibility_1}
- {responsibility_2}

---

## Absolute Boundaries

> **NEVER** violate these boundaries under any circumstances.

- **[{persona.name}]** NEVER: {boundary_1}
- **[{persona.name}]** NEVER: {boundary_2}

---

## Governance Rules

| Rule | Gate | Authority | On Violation |
|------|------|-----------|--------------|
| {gate.description} | {gate.id} | {authority_level} | {violation_policy} |

> When a STRICT rule is violated: **STOP and ask for human review.**

---

## Output Targets

| Persona | Target |
|---------|--------|
| {persona.name} | {persona.output_target} |

---

## Capabilities

| Capability | Owner | Description |
|-----------|-------|-------------|
| {cap.id} | {cap.owner_persona} | {cap.description} |
```

### 2.3 类设计

```python
class ClaudeCodeAdapter(BaseAdapter):
    """Translates PGCDocument into Claude Code's CLAUDE.md format."""

    def render(self, document: PGCDocument) -> Dict[str, str]:
        """Render PGCDocument to CLAUDE.md."""
        sections = [
            self._render_header(),
            self._render_roles(document.personas),
            self._render_boundaries(document.personas),
            self._render_governance(document),
            self._render_output_targets(document.personas),
            self._render_capabilities(document.capabilities),
        ]
        return {"CLAUDE.md": "\n\n".join(sections)}

    def get_target_runtime(self) -> str:
        return "claude-code"

    def get_supported_extensions(self) -> List[str]:
        return [".md"]

    # Private rendering methods
    def _render_header(self) -> str: ...
    def _render_roles(self, personas) -> str: ...
    def _render_boundaries(self, personas) -> str: ...
    def _render_governance(self, document) -> str: ...
    def _render_output_targets(self, personas) -> str: ...
    def _render_capabilities(self, capabilities) -> str: ...
```

### 2.5 设计决策

| 决策 | 选项 | 选择 | 理由 |
|------|------|------|------|
| 是否生成 mcp_tools_schema.json | 是 / 否 | 否 | Issue 标记为可选，先聚焦 CLAUDE.md 核心功能 |
| violation policy 来源 | 从 governance_authority 推断 / 硬编码 | 从 governance_authority 推断 | PGC 文档已声明 violation policy，应利用 |
| 空 boundaries 是否渲染章节 | 是 / 否 | 否 | 避免空章节，减少噪音 |
| 多 Persona 排序 | 声明顺序 / 字母序 | 声明顺序 | 保持与源文档一致 |
