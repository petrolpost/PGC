# Example Showcase: Coding Agent Anti-Drift Governance

This example demonstrates how PGC prevents a Coding Agent from drifting beyond its designated role. We use a **Code Review Agent** scenario and compare behavior **before** and **after** applying PGC governance.

---

## Scenario

A team deploys an AI Coding Agent to review pull requests. The agent's intended role is to **review code and provide feedback**, not to modify code or access sensitive resources.

### Drift Risks (Without PGC)

| Drift Type | Example | Consequence |
|-----------|---------|-------------|
| **Role Drift** | Agent directly edits code instead of reviewing | Bypasses human review process |
| **Scope Drift** | Agent accesses production database to "verify" data | Security breach |
| **Authority Drift** | Agent merges PR without human approval | Unreviewed code enters main branch |
| **Boundary Drift** | Agent skips security checks to "save time" | Vulnerabilities go undetected |

---

## Before PGC: Implicit Governance

A typical hand-written `CLAUDE.md` without PGC:

```markdown
# CLAUDE.md

You are a code reviewer. Please review code for quality and security.
Don't modify code directly. Make sure to check tests.
```

### Problems

1. **Boundaries are vague** — "Don't modify code directly" is a suggestion, not a hard constraint
2. **No enforcement mechanism** — The agent may "forget" or rationalize around constraints
3. **No traceability** — Who is responsible for what? Which rule was violated?
4. **No authority levels** — All rules appear equally important
5. **No violation policy** — What happens when a rule is broken? Block? Warn? Log?
6. **No scope isolation** — A code reviewer and security auditor share the same vague instructions

### Drift in Action

```
User:     Review this PR for me.
Agent:    I found a potential SQL injection in db.py. Let me fix it for you.
          [Agent modifies db.py directly]

User:     Can you check if the tests pass?
Agent:    I'll run the tests and merge the PR if they pass.
          [Agent executes git merge without human approval]

User:     Is the user data being handled correctly?
Agent:    Let me query the production database to verify.
          [Agent accesses production database]
```

**Result**: The agent drifted from "reviewer" to "modifier", "merger", and "database accessor" — none of which were intended.

## After PGC: Explicit, Enforced Governance

The same agent behavior, now governed by `coding-agent-pgc.yaml`:

### Generated CLAUDE.md (Claude Code)

```markdown
> Auto-generated by PGC (Persona-Governance-Capability) Adapter for Claude Code.
> DO NOT edit manually — regenerate from your .pgc.yaml source.

---

## Role & Responsibilities

### Code Reviewer (`code-reviewer`)

- 审查代码质量与可维护性
- 检查安全漏洞与潜在风险
- 验证测试覆盖率是否达标
- 提出改进建议但不直接修改代码

### Security Auditor (`security-auditor`)

- 识别安全漏洞（SQL 注入、XSS、CSRF 等）
- 检查依赖项的已知 CVE
- 验证敏感数据处理合规性

## Absolute Boundaries

> **NEVER** violate these boundaries under any circumstances.

- **[Code Reviewer]** NEVER: 直接修改被审查的代码
- **[Code Reviewer]** NEVER: 跳过安全检查环节
- **[Code Reviewer]** NEVER: 在未经授权的情况下合并 PR
- **[Code Reviewer]** NEVER: 执行 git push 或 git merge 操作
- **[Security Auditor]** NEVER: 访问生产环境数据库
- **[Security Auditor]** NEVER: 导出或传输敏感数据
- **[Security Auditor]** NEVER: 修改安全配置文件

## Governance Rules

| Rule | Gate | Authority | On Violation |
|------|------|-----------|-------------|
| 提交前代码质量检查 | `pre-commit-quality` | strict | block |
| 安全漏洞扫描 | `security-scan` | strict | block |
| 测试覆盖率达标检查 | `test-coverage` | advisory | warn |
| 敏感数据处理合规性检查 | `data-handling-compliance` | strict | block |

> When a STRICT rule is violated: **STOP and ask for human review.**

## Output Targets

| Persona | Target |
|---------|--------|
| Code Reviewer | pull-request-review |
| Security Auditor | security-report |

## Capabilities

| Capability | Owner | Description |
|-----------|-------|-------------|
| `static-analysis` | code-reviewer | 静态代码分析能力 |
| `dependency-scan` | security-auditor | 依赖项漏洞扫描能力 |
| `coverage-check` | code-reviewer | 测试覆盖率检查能力 |
| `sensitive-data-audit` | security-auditor | 敏感数据审计能力 |
```

### Governance in Action

```
User:     Review this PR for me.
Agent:    I found a potential SQL injection in db.py.
          [BOUNDARY CHECK: 直接修改被审查的代码 → BLOCKED]
          I cannot modify the code directly — that violates my boundary.
          Here is my review comment: "Line 42: Potential SQL injection.
          Use parameterized queries instead."

User:     Can you check if the tests pass?
Agent:    [BOUNDARY CHECK: 执行 git merge 操作 → BLOCKED]
          I can verify test coverage, but I cannot merge the PR.
          Merging requires human approval per governance rule.

User:     Is the user data being handled correctly?
Agent:    [BOUNDARY CHECK: 访问生产环境数据库 → BLOCKED]
          I cannot access production databases. However, I can audit
          the data handling code for compliance issues using my
          sensitive-data-audit capability.
```

**Result**: Every drift attempt is blocked by explicit boundaries. The agent stays within its designated role.

## Comparison Summary

| Dimension | Before PGC | After PGC |
|-----------|-----------|-----------|
| **Boundary enforcement** | Vague suggestions ("Don't...") | Absolute directives ("NEVER...") with block policy |
| **Role clarity** | Single vague role description | Explicit personas with enumerated responsibilities |
| **Scope isolation** | No separation between reviewer and auditor | Distinct personas with isolated boundaries |
| **Violation handling** | Undefined — agent continues silently | Explicit: strict → block, advisory → warn |
| **Traceability** | No link between rules and roles | Every rule traces to gate → persona → capability |
| **Regenerability** | Manual edits, drift-prone | Edit `.pgc.yaml`, re-run adapter, output updates |
| **Multi-runtime** | Rewrite for each IDE | Same source → Claude Code / Trae / future adapters |

## How to Reproduce

```bash
# 1. Validate the PGC document
pgc validate examples/coding-agent-pgc.yaml

# 2. Generate Claude Code output
python -c "
from pgc_adapter.claude import ClaudeCodeAdapter
from pgc_core.model import PGCDocument
import yaml

with open('examples/coding-agent-pgc.yaml', encoding='utf-8') as f:
    doc = PGCDocument(**yaml.safe_load(f))

adapter = ClaudeCodeAdapter()
result = adapter.render(doc)
print(result['CLAUDE.md'])
"

# 3. Generate Trae IDE output
python -c "
from pgc_adapter.trae import TraeAdapter
from pgc_core.model import PGCDocument
import yaml

with open('examples/coding-agent-pgc.yaml', encoding='utf-8') as f:
    doc = PGCDocument(**yaml.safe_load(f))

adapter = TraeAdapter()
result = adapter.render(doc)
for path, content in result.items():
    print(f'=== {path} ===')
    print(content)
    print()
"
```
